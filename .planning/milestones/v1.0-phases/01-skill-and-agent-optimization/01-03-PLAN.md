Two changes are needed:

**Change 1: Add --minimal flag parsing in gsd-tools.cjs**

In `~/.claude/get-shit-done/bin/gsd-tools.cjs`, find the `case 'generate-claude-md':` block (around line 877). Add `minimalFlag` parsing:

```javascript
    case 'generate-claude-md': {
      const outputIdx = args.indexOf('--output');
      const outputPath = outputIdx !== -1 ? args[outputIdx + 1] : null;
      const autoFlag = args.includes('--auto');
      const forceFlag = args.includes('--force');
      const minimalFlag = args.includes('--minimal');
      profileOutput.cmdGenerateClaudeMd(cwd, { output: outputPath, auto: autoFlag, force: forceFlag, minimal: minimalFlag }, raw);
      break;
    }
```

The only addition is the `minimalFlag` line and passing `minimal: minimalFlag` in the options object.

**Change 2: Handle minimal mode in cmdGenerateClaudeMd function in profile-output.cjs**

In `~/.claude/get-shit-done/bin/lib/profile-output.cjs`, find the `cmdGenerateClaudeMd` function (around line 817). Modify it to skip project, stack, conventions, and architecture sections when `options.minimal` is true.

The approach: When `options.minimal` is true, replace the content of the 4 sections (project, stack, conventions, architecture) with a single line: `Loaded on demand by GSD commands.`

This preserves the marker-bounded update mechanism (markers stay, content is minimal).

Find the loop `for (const name of MANAGED_SECTIONS)` (around line 839) and add a conditional:

```javascript
  for (const name of MANAGED_SECTIONS) {
    // In minimal mode, skip heavy sections - keep markers but use placeholder content
    if (options.minimal && ['project', 'stack', 'conventions', 'architecture'].includes(name)) {
      const heading = sectionHeadings[name];
      generated[name] = {
        content: 'Loaded on demand by GSD commands.',
        source: 'minimal',
        hasFallback: false,
      };
      sectionsGenerated.push(name);
      continue;
    }
    const gen = generators[name](cwd);
    generated[name] = gen;
    if (gen.hasFallback) {
      sectionsFallback.push(name);
    } else {
      sectionsGenerated.push(name);
    }
  }
```

The key change is the `if (options.minimal ...)` block that replaces the generator output with a placeholder for the 4 heavy sections. The workflow section is NOT in the skip list, so it generates normally.

**What NOT to change:**
- Do not modify `generateProjectSection`, `generateStackSection`, etc. -- they are still needed for `--full` mode
- Do not remove any markers -- the marker-bounded update mechanism must continue to work
- Do not change the default behavior (without --minimal flag, generate-claude-md works exactly as before)
