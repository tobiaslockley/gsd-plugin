Note on output pattern: The cmd* functions call `output(result)` which writes JSON to stdout and exits.
The MCP server MUST NOT call these directly — it must either:
(a) import the internal helper functions that return values (e.g., findPhaseInternal, getRoadmapPhaseInternal), or
(b) wrap cmd* calls to capture their output before they exit the process.

Approach (a) is strongly preferred. The *Internal functions (findPhaseInternal, getRoadmapPhaseInternal, etc.) return data without side effects. For state operations, use safeReadFile + manual parsing, or refactor to extract the data-returning portion.

    **Step 2: Create server.cjs**

    Create `~/.claude/get-shit-done/mcp/server.cjs` implementing an MCP server with stdio transport (per D-01).

    The server MUST:

    1. Import from `@modelcontextprotocol/sdk/server/index.js` — the MCP SDK classes `McpServer` (or `Server`) and `StdioServerTransport` (from `@modelcontextprotocol/sdk/server/stdio.js`). Check the SDK's actual export paths after `npm install` — the SDK may use `/dist/` paths or different entry points. Read `node_modules/@modelcontextprotocol/sdk/package.json` exports field to confirm.

    2. Import GSD lib modules using relative paths:
       ```javascript
       const core = require('../bin/lib/core.cjs');
       const { extractFrontmatter } = require('../bin/lib/frontmatter.cjs');
       ```

    3. Determine project root: The MCP server receives the project working directory. Use `core.findProjectRoot(process.cwd())` or accept it from the server arguments. If no `.planning/` directory exists, the server should still start but return empty/error responses for resources.

    4. Register these MCP resources (per D-02 — read-only project state):

       **`gsd://state`** — Current project state
       - Read `.planning/STATE.md` via `core.safeReadFile()`
       - Parse frontmatter with `extractFrontmatter()`
       - Return JSON with: `status`, `stopped_at`, `last_updated`, `progress` (from frontmatter), plus key body sections (Current Position, Decisions, Blockers)

       **`gsd://roadmap`** — Full roadmap overview
       - Read `.planning/ROADMAP.md` via `core.safeReadFile()`
       - Return the full content as text (roadmap is already a good summary)

       **`gsd://requirements`** — Requirements status
       - Read `.planning/REQUIREMENTS.md` via `core.safeReadFile()`
       - Return full content as text

       **`gsd://config`** — Project configuration
       - Use `core.loadConfig(projectRoot)` to get parsed config
       - Return JSON with all config values

       **`gsd://phase/{phase_number}`** — Phase info by number
       - Use `core.findPhaseInternal(projectRoot, phaseNumber)` to locate phase dir
       - Use `core.getRoadmapPhaseInternal(projectRoot, phaseNumber)` to get roadmap section
       - Return JSON with: phase directory path, plan count, summary count, roadmap section, has_research, has_context flags

       **`gsd://phase/{phase_number}/context`** — Phase CONTEXT.md content
       - Locate phase dir via `core.findPhaseInternal()`
       - Read `{phase_dir}/{padded}-CONTEXT.md` if it exists
       - Return content as text, or error if not found

    5. Use the MCP SDK's resource registration API. The exact API depends on the SDK version. After installing, read the SDK source to confirm the registration pattern. Expected pattern:
       ```javascript
       server.resource("state", "gsd://state", async (uri) => { ... });
       ```
       Or using the lower-level `server.setRequestHandler(ListResourcesRequestSchema, ...)` pattern.

    6. Error handling: Do NOT use `core.error()` (it calls `process.exit(1)`). Instead, catch errors and return MCP error responses. Wrap all file reads in try/catch. If `.planning/` does not exist, return a helpful message: "No GSD project found. Run /gsd:new-project to initialize."

    7. Server startup: Connect via stdio transport at the end of the file:
       ```javascript
       const transport = new StdioServerTransport();
       await server.connect(transport);
       ```
       Wrap in an async IIFE since this is CommonJS.

    8. Server metadata: Name the server "gsd" with version "1.0.0" and capabilities: `{ resources: {} }` (tools added in Plan 02).
