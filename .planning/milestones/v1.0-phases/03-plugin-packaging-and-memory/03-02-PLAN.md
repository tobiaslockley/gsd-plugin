<objective>
Migrate GSD command, skill, agent, template, and reference content from the legacy home-directory layout into the repo-owned plugin layout, eliminating `execution_context` indirection and hardcoded legacy paths.

Purpose: Packaging only works if the content itself is portable. This plan converts the actual GSD content surface so plugin installs carry their own workflows, agents, templates, and references.

Output: skills, agents, templates, and references live in the plugin repo layout and no workflow-critical content depends on `~/.claude/get-shit-done`.
</objective>
